## 作るもの: 集中ポモドーロ物理タイマー

サーボの針が残り時間を物理的に示し、終わったらブザーで知らせるデスクガジェット。
**買い足しゼロ円**、上の部品だけで完成します。

- ボタン短押し → 25分の集中スタート（針がゆっくり0へ）
- 終わるとブザー → 自動で5分休憩 → またブザー → 待機に戻る
- 動作中の短押し = ポーズ/再開、長押し（約1秒）= リセット

スケッチ: [pomodoro_timer/pomodoro_timer.ino](pomodoro_timer/pomodoro_timer.ino)

### 配線（はんだ付け不要・ブレッドボードに挿すだけ）

#### まず「レール」を理解する

ブレッドボードの**左右の端にある、赤い線と青い線がついた長い列**を「**電源レール**」と呼びます。
**同じレールの穴はすべて内部でつながっている**ので、電源とGNDの「共有スポット」を作れます。

```
赤い線 → + + + + + + + + + +   ← 赤レール（プラス／5V を流す列）
青い線 → − − − − − − − − − −   ← 青レール（マイナス／GND を流す列）
        ［中央のたくさんの穴に部品を挿す］
```

今回はGNDを使う部品が**サーボ・ブザー・ボタンの3つ**あります。Arduinoのピンに直接挿すと
足りないので、**Arduinoの GND を青レールに1本、5V を赤レールに1本**だけつなぎ、
残りの部品はそのレールに挿して共有します。

#### 大事な考え方: 部品はArduinoに直接つながない

部品の足は短く固いので、Arduinoのピンに直接は挿せません。**いったん全部ブレッドボードに
挿し、そこからジャンパー線（コネクタ付ケーブル オスオス）でArduinoのピンへつなぎます。**

```
  部品 → ブレッドボードに挿す → ジャンパー線 → Arduinoのピン
```

ブレッドボードの**中央の溝をはさんだ縦5穴は内部でつながっています**（溝の左右は別グループ）。
この「つながった列」を中継地点として使います。

```
     a b c d e │ f g h i j   ← 縦5穴ずつが内部でつながっている
   1 ○ ○ ○ ○ ○ │ ○ ○ ○ ○ ○      (a1〜e1 が同じ列)
   2 ○ ○ ○ ○ ○ │ ○ ○ ○ ○ ○      中央の溝 │ の左右は別々
```

#### Step 1: まずレールへ電源を1本ずつ（USBは抜いた状態で）

| Arduino のピン | つなぎ先          | 説明                         |
| -------------- | ----------------- | ---------------------------- |
| **5V**         | ブレッドボード赤レール | 5Vの共有スポットを作る   |
| **GND**        | ブレッドボード青レール | GNDの共有スポットを作る  |

> ピンの位置（USBを上にして見たとき）
> - **5V** … 基板**左側**の `+5V OUT` と書かれた赤いピン
> - **GND** … `+5V` のすぐ下に GND が2つ。どちらでもOK

#### Step 2: 各部品を挿してジャンパー線でつなぐ

「Arduino **D9**」などは、**部品を挿した列からジャンパー線でそのピンへつなぐ**という意味です
（部品を直接ピンに挿すわけではありません）。

| 部品                | つなぎ先（→ジャンパー線で） | Arduino上の位置                  |
| ------------------- | --------------------------- | -------------------------------- |
| サーボ 黄線（信号） | Arduino **D9**              | 右側の列、`~D9`                  |
| サーボ 赤線（電源） | **赤レール**                | （= 5Vを共有）                   |
| サーボ 茶線（GND）  | **青レール**                | （= GNDを共有）                  |
| ブザー 片方         | Arduino **D8**              | 右側の列、`~D9` の真下の `D8`    |
| ブザー もう片方     | **青レール**                | （= GNDを共有）                  |
| ボタン 片足         | Arduino **D2**              | 右側の列、下のグループの `D2`    |
| ボタン 対角の足     | **青レール**                | （= GNDを共有）                  |

> サーボはコネクタ付きなので、ジャンパー線（オスオス）で D9 / 赤レール / 青レール へ伸ばします。

**🔘 ボタン（タクトスイッチ）の挿し方**

4本足のうち**対角の2本**を使います。**中央の溝をまたぐように挿す**と、つながった足のペアが
左右に分かれて対角を選びやすくなります。

```
     a b c d e │ f g h i j
   5 ○ ○[足]○ ○│○ ○[足]○ ○     ← 4本足が溝をまたぐ
   6 ○ ○[足]○ ○│○ ○[足]○ ○

   左側の足の列 ── ジャンパー線 ──→ Arduino D2
   対角の足の列 ── ジャンパー線 ──→ 青レール(GND)
```

> 足が広がって挿しにくければ、**指で軽くまっすぐ曲げて整えてOK**（折れるので曲げは一度で）。

**🔊 ブザー（圧電サウンダ）の挿し方**

2本足を**別々の縦列**に挿すだけ（向きは自由）。足は細めなので穴の間隔に合わせて軽く広げます。

```
   片方の足の列 ── ジャンパー線 ──→ Arduino D8
   もう片方の足の列 ── ジャンパー線 ──→ 青レール(GND)
```

> サーボの信号線はメーカーにより**橙または黄色**。本キットは黄色。
> ボタンに抵抗は不要（コード側の `INPUT_PULLUP` で代用）。
> ブザーの向きは気にしなくてOK。**配線は必ずUSBを抜いた状態で**行う。

#### 全体図

```
        Arduino Uno（USBを上にして右側の列）
   D9 ──────────────── サーボ 黄(信号)
   D8 ──────────────── ブザー 片方
   D2 ──────────────── ボタン 片足

   5V ──── 赤レール ── サーボ 赤
  GND ──── 青レール ──┬─ サーボ 茶
                      ├─ ブザー もう片方
                      └─ ボタン 対角の足
```

### 書き込み手順

1. [arduino.cc](https://www.arduino.cc/en/software) から **Arduino IDE** を入れる。
2. Uno を USB(A-B) でPCに接続。
3. IDE の「ツール」→ ボード=**Arduino Uno**、ポート=`/dev/cu.usbmodem...` を選択。
4. `pomodoro_timer/pomodoro_timer.ino` を開き、**✓検証** → **→書き込み**。
5. 「Done uploading」かつ **針がスタート位置に動けば成功**。

### 文字盤の作り方（見た目の肝）

1. サーボの白い腕（ホーン）に、軽い紙やつまようじを貼って**針**にする。
2. 紙に半円を描き、目盛り `25 20 15 10 5 0` と、0側に「BREAK」ゾーンを記入。
3. サーボを机にマスキングテープで仮固定し、針が目盛りを指すよう向きを合わせる。

### 動作確認のコツ

スケッチ先頭の `TEST_MODE`：

- `true`（初期値）… 25秒/5秒で1サイクルを素早く確認。
- `false` … 本番の25分/5分。

シリアルモニタ（**9600 baud**）に状態と残り秒数が出ます。

## 振り子メトロノーム

サーボの針が本物の振り子のように左右へ滑らかに往復し、1拍ごとにブザーが鳴る電子メトロノーム。
**配線はポモドーロタイマーと全く同じ**で、書き込むスケッチを入れ替えるだけ。

スケッチ: [metronome/metronome.ino](metronome/metronome.ino)

- 起動すると **100 BPM** で自動再生。針が左右に振れ、拍ごとに「ピッ」（**小節の1拍目だけ高音アクセント**、4/4拍子）。
- **短押し = タップテンポ**: 一定のリズムで2〜数回叩くと、その間隔から BPM を自動計算（直近4回の平均で安定化）。
- **長押し（約1秒）= 再生/停止**: 停止中は針が中央で静止。

### 調整できる場所（スケッチ先頭の定数）

| 定数 | 役割 | 例 |
| --- | --- | --- |
| `BEATS_PER_BAR` | 拍子 | `3` でワルツ（3拍子） |
| `SWING_MIN_ANGLE` / `SWING_MAX_ANGLE` | 針の振り幅 | 広げると大きく振れる |
| `ACCENT_FREQ` / `BEAT_FREQ` | 音程 | アクセント音／通常音の周波数 |
| `DEFAULT_BPM` | 起動時のテンポ | `120` など |

### こだわりポイント

- 針は**コサインのイージング**で端に近づくと減速する自然な振り子の動き。
- `tone()` の自動停止を使った**ノンブロッキング設計**で、演奏中もボタンが効く。

## サーボ単体テスト（診断ツール）

配線・電源・信号（D9）が正常かを、本体ロジックから切り離して確認するためのスケッチ。
**「針が動かない」ときの原因切り分け**に使う。

スケッチ: [servo_test/servo_test.ino](servo_test/servo_test.ino)

- 書き込むとサーボが **0° → 90° → 180°** を1秒ごとに往復し続ける。
- これが動けば「サーボ・配線・電源・信号」はすべて正常 → 原因は本体スケッチ側かボタン等。
- これも動かなければ、サーボの電源・配線を疑う。

## トラブルシューティング（実際にハマった点）

| 症状 | 原因 | 対処 |
| --- | --- | --- |
| 書き込みが `cannot open port ... No such file or directory` で失敗 | **ポート番号がズレている**（抜き差しで `usbmodem1201`→`1301` のように変わる） | 「ツール → ポート」で現在のポートを選び直す。書き込み前にシリアルモニタは閉じる |
| 針が全く動かない・ボタンが効かない | **部品の半挿し（接触不良）**。特にタクトスイッチ | ブレッドボードの奥までしっかり挿し直す。サーボ単体テストで切り分ける |
| `Done compiling` は出るのに動かない | **コンパイル成功と書き込み成功は別**。書き込み（→ボタン）まで完了しているか確認 | `Done uploading.` が出ているか確認する |
